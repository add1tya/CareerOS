# assets/

## Purpose

Stores static, non-code resources shared across apps, services, and documentation.

## What belongs here

- Brand assets (logos, icons, wordmarks)
- Illustrations and marketing imagery
- Fonts and typographic resources
- Audio or video files used in the product
- Shared media referenced by multiple apps or docs

## What should never be placed here

- Source code of any kind (use `apps/`, `services/`, `packages/`)
- Documentation in markdown or other text formats (use `docs/`)
- AI prompt templates (use `prompts/`)
- Configuration files or environment secrets
- Generated build artifacts
- Scripts or automation tooling (use `scripts/`)
