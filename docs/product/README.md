# docs/product/

## Purpose

Defines what CareerOS is, who it serves, and the principles that guide product decisions. This is the authoritative product layer — implementation in `apps/` and `services/` must align with documents here.

## What belongs here

- Product vision and long-term direction
- Product philosophy and positioning
- Guiding principles for feature and UX decisions
- Glossary of domain terms and product language
- User personas, journeys, and success criteria (when documented)
- Feature specifications before they move to architecture or API design

## What should never be placed here

- System architecture diagrams or service topology (use `docs/architecture/`)
- Database schemas or ER diagrams (use `docs/database/`)
- API endpoint specifications (use `docs/api/`)
- AI model prompts or inference pipelines (use `docs/ai/` and `prompts/`)
- UI mockups, component specs, or design tokens (use `docs/ui/`)
- Engineering trade-off records (use `docs/decisions/`)
