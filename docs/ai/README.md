# docs/ai/

## Purpose

Documents the AI systems that power CareerOS — how intelligence is designed, orchestrated, evaluated, and integrated into the product experience.

## What belongs here

- AI system architecture and component responsibilities
- Model selection criteria and usage guidelines
- Orchestration flows (e.g., planning, mentorship, accountability)
- Evaluation methodology and quality benchmarks
- Safety, guardrails, and human-in-the-loop patterns
- Context management and memory design concepts

Runtime prompt content lives in `prompts/`; this directory explains the systems that consume those prompts.

## What should never be placed here

- Product vision or positioning (use `docs/product/`)
- General backend service topology unrelated to AI (use `docs/architecture/`)
- Non-AI database schemas (use `docs/database/`)
- HTTP API endpoint specs (use `docs/api/`)
- Visual design guidelines (use `docs/ui/`)
- ADRs for non-AI engineering decisions (use `docs/decisions/`)
