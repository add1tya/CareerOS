# docs/ui/

## Purpose

Defines the user experience layer of CareerOS — how users interact with the system across web, mobile, and future client surfaces.

## What belongs here

- Design system principles and visual language
- Component specifications and interaction patterns
- Accessibility standards and responsive behavior guidelines
- Navigation and information architecture
- UX flows for documented product features
- Platform-specific considerations (web vs. mobile)

## What should never be placed here

- Product vision or business requirements (use `docs/product/`)
- Backend or service architecture (use `docs/architecture/`)
- Database or API specifications (use `docs/database/`, `docs/api/`)
- AI orchestration or model design (use `docs/ai/`)
- Executable frontend source code (use `apps/`)
- Architecture Decision Records (use `docs/decisions/`)
